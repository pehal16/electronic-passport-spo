# Пользовательские сценарии (MVP)

## 1. Сценарий: Куратор открывает паспорт группы

1. Куратор входит в систему.
2. На главной выбирает свою группу.
3. Открывает вкладки паспорта: состав, посещаемость, успеваемость, риски.
4. Видит проблемных студентов и последние уведомления.

Результат: куратор получает целостную картину группы за 1-2 минуты.

## 2. Сценарий: Преподаватель отмечает посещаемость

1. Преподаватель открывает занятие из расписания.
2. Система показывает список студентов группы.
3. Преподаватель отмечает статус: присутствовал/отсутствовал/опоздал.
4. Нажимает «Сохранить».

Результат: журнал посещаемости заполнен за 1 минуту.

## 3. Сценарий: Преподаватель выставляет оценки

1. Преподаватель открывает урок.
2. Вводит оценки и комментарии по студентам.
3. Отмечает долги (если есть).
4. Сохраняет журнал.

Результат: оценки отображаются студенту, куратору и родителю (по политике доступа).

## 4. Сценарий: Куратор видит риск по студенту

1. Система автоматически формирует сигнал риска по порогам.
2. Куратор получает карточку риска с причиной.
3. Куратор фиксирует действие: беседа, звонок родителю, контроль.

Результат: есть подтвержденная реакция на риск и история работы.

## 5. Сценарий: Студент проверяет учебный статус

1. Студент входит в систему.
2. На главной видит расписание, новые оценки, пропуски и долги.
3. Открывает детали по конкретной дисциплине.

Результат: студент понимает, что нужно закрыть в первую очередь.

## 6. Сценарий: Родитель получает уведомление

1. Куратор отправляет уведомление родителям группы или выбранного студента.
2. Родитель открывает кабинет и видит сообщение.
3. Нажимает «Ознакомлен».

Результат: фиксируется факт ознакомления и время.

## 7. Сценарий: Администратор настраивает доступ

1. Администратор создает пользователя.
2. Назначает роль и привязку к группе/дисциплине.
3. Проверяет доступ через аудит или тестовый вход.

Результат: новый пользователь получает только нужные права.

## 8. Сценарий: Работа с документами группы

1. Куратор загружает документ в паспорт группы.
2. Выбирает категорию и уровень видимости.
3. Система показывает документ только тем ролям, которым разрешен доступ.

Результат: единый репозиторий документов без лишней видимости.

## 9. Сценарий: Сводка для администрации

1. Пользователь роли management открывает дашборд.
2. Выбирает период и группы.
3. Получает сводку по посещаемости, успеваемости и рискам.

Результат: управленческая картина без доступа к лишним персональным данным.
